# Manuscript

This folder is the source of truth for Volume I of *The Architecture of Dependence*.

## Editorial Voice

The manuscript should read as though the reader is discovering each question for the first time.

Avoid:

- retrospective narration
- autobiographical commentary
- explaining why a question matters before asking it

Prefer:

- direct questions
- short reflective paragraphs
- one central question per chapter
- chapters ending by naturally generating the next question

Every chapter should feel inevitable.

# Editorial Principles

The purpose of this project is not to produce a philosophical system.

The purpose is to faithfully reconstruct an enquiry.

Therefore:

• Preserve every genuine question.

• Improve language without changing philosophy.

• Never invent conclusions.

• Traditions answer the questions.

They never become the subject of the book.

• Questions choose the participants.

• Every explanation must answer the same test:

What explanatory work does this concept actually perform?

• An unanswered question honestly preserved is preferable to an answer accepted prematurely.
